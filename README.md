# CS-330 Portfolio Reflection

## How do I approach designing software?
I approach designing software by first understanding the problem and identifying the main goals of the application. For this project, I focused on how each object in the scene would behave and interact before writing code. Breaking the scene into smaller components, such as shapes, movement, and interactions, helped me stay organized and build the project step by step.

## What new design skills did this project help me develop?
This project helped me strengthen my ability to design visual layouts and think in terms of spatial relationships. I learned how color, positioning, and object behavior affect user engagement in a 3D scene.

## What design process did I follow?
I followed an iterative design process. I started with a basic scene and gradually added features such as textures, movement, and collision behavior. Testing frequently allowed me to catch issues early and refine the design.

## How do I approach developing programs?
I approach development by writing small, testable sections of code and building on them. This project required careful debugging and incremental improvements.

## How has this project helped me reach my goals?
Computational graphics and visualization expanded my understanding of how math, logic, and programming come together to create interactive visuals. These skills are valuable for both my future coursework and professional goals in computer science.
